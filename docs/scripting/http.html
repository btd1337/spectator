# HTTP

The HTTP object provides methods to make additional requests during your Spectator request operation. The objects methods are the same as HTTP methods.

Note: The corresponding HTTP delete method is named destroy because delete is a keyword in JavaScript.

All methods take a mandatory URL string as first parameter and an optional config object as second parameter.

When the HTTP request is finished it returns either a response object or undefined if the request failed.

# get(url[, config])

url A valid URI string. If the URI is not valid the request is aborted

config An object, see below for more

returns a response object or undefined if the request failed

# post(url[, config])

url A valid URI string. If the URI is not valid the request is aborted

config An object, see below for more

returns a response object or undefined if the request failed

# put(url[, config])

url A valid URI string. If the URI is not valid the request is aborted

config An object, see below for more

returns a response object or undefined if the request failed

# patch(url[, config])

url A valid URI string. If the URI is not valid the request is aborted

config An object, see below for more

returns a response object or undefined if the request failed

# destroy(url[, config])

url A valid URI string. If the URI is not valid the request is aborted

config An object, see below for more

returns a response object or undefined if the request failed

Example:

function before_sending(request) {
  var res = HTTP.get("https://httpbin.org/get");
}

# Config Object for HTTP

The config object configures the HTTP requests. It consists only of fields which are all optional to set.

Note: As the HTTP requests are not concurrent right now it is recommended to set at least the timeout field so your application won't freeze to death.

# headers

headers is of type object

The headers field is an flat object which currently consists of string-string pairs. All headers defined here will be added the the HTTP request

# timeout

timeout is of type string or number

Sets the timeout for the HTTP request

# body

body is of type string or object Sets the HTTP body for the request. This field will be ignored when the HTTP method has no body (e.g. GET).

If the type of the body value is a string, the string will be sent as is. By default the content-type will be set to text/plain.

If the users specifies a different content-type in the header section, this value will be taken.

When the type of the body value is an object, it will be serialized to JSON and sent with the content-type application/json.

The two helper functions UrlEncoded and FormData are converting the objects to an url encoded or form data format.

Example:

function before_sending(request) {
  var res1 = HTTP.post("http://localhost:8000/login", {
    "headers": {
      "foo:" "bar"
    }
    "timeout": 10,
    "body": {
      "foo": "baz",
      "hello": "there"
    }
  });

  var res2 = HTTP.post("http://localhost:8000/foo", {
    "body": UrlEncoded({"parse": "this!"})
  });

  var res3 = HTTP.post("http://localhost:8000/foo", {
    "body": FormData({"For a": "Form"})
  });

  var res4 = HTTP.post("http://localhost:8000/xml", {
    "headers": {"Content-Type": "application/xml"}
    "body": "<someXml></someXml>"
  });
}
